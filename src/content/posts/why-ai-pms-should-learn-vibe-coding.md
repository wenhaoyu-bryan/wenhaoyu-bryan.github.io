## The PM Bottleneck Has Shifted

Traditional PMs wait for engineering to build, then review. AI PMs who vibe code can validate product ideas in hours instead of weeks. The bottleneck isn't ideas — it's the ability to turn ideas into something testable.

## What Is Vibe Coding for PMs?

Vibe coding is using LLM-based tools like Claude Code or Cursor to build functional prototypes through natural language conversation. You define the product intent, architecture constraints, and design boundaries. The LLM generates code.

For PMs, this isn't about becoming an engineer. It's about three specific capabilities:

1. **Faster validation.** You can test whether a product idea makes sense before committing engineering resources.
2. **Better specs.** When you've built a prototype, your PRDs become more concrete and technically grounded.
3. **Closer engineering collaboration.** You can speak in architecture patterns, not just user stories.

## What I've Learned

Building [Prompt-to-Ontology](/projects/prompt-to-ontology) entirely through vibe coding taught me that the most valuable PM output isn't the code — it's the ability to define clear architecture boundaries, specify data flow, and articulate what "done" looks like.

The LLM handles implementation. The PM handles product thinking. The overlap is where the magic happens.

## Getting Started

You don't need to be a software engineer. You need to be a PM who can:

- Define clear architecture constraints
- Specify data models and API contracts
- Describe user flows in technical terms
- Review generated code for product correctness

The tools are ready. The question is whether you are.
